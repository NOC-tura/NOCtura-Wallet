# Security Policy

## Reporting Security Vulnerabilities

We take security very seriously. If you discover a security vulnerability, please report it responsibly.

**DO NOT** open a public issue for security vulnerabilities.

Instead, email us at: **security@noctura.io**

Please include:
1. Description of the vulnerability
2. Steps to reproduce (if applicable)
3. Potential impact
4. Suggested fix (if you have one)

We will respond to legitimate security reports within 48 hours and work with you on a fix.

## Security Practices

### Code Review
- All code changes undergo peer review
- Security-critical code receives additional scrutiny
- Automated security scanning is in place

### Dependencies
- Regular updates and vulnerability scanning
- Dependabot alerts are monitored
- Minimal dependency footprint

### Cryptography
- Uses audited cryptographic libraries
- No custom cryptographic implementations
- Regular security audits of cryptographic code

### Testing
- Comprehensive test coverage
- Fuzz testing for cryptographic functions
- Regular penetration testing
